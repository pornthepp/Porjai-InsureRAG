# Hybrid Router Design

## เป้าหมาย

ลดค่าเรียก LLM แต่ยังเข้าใจคำถามประกันที่ไม่มีคำตรงในรายการ keyword เช่น `ประกัน 2+ กับ 3+ ต่างกันยังไง` และ `ซ่อมห้างกับซ่อมอู่ต่างกันยังไง`

## ทางเลือก

1. Keyword อย่างเดียว: ฟรีและเร็ว แต่พลาดคำใหม่หรือภาษาลูกค้าได้ง่าย
2. LLM ทุกคำถาม: เข้าใจภาษาดีกว่า แต่เสียค่า API ตลอด
3. Hybrid: ใช้ keyword ก่อน แล้วเรียก LLM เฉพาะตอนที่ยังแยกหมวดไม่ได้

เลือกข้อ 3 เพราะสมดุลค่าใช้จ่ายกับความแม่นยำที่สุดสำหรับโปรเจกต์นี้

## ลำดับการทำงาน

1. Offline Gate ปิดคำทักทายและคำถามนอกเรื่องโดยไม่เรียก LLM
2. คำสั้นหรือคำถามกว้าง เช่น `ประกัน`, `รถยนต์`, `ประกันชีวิต` ตอบถามรายละเอียดเพิ่มโดยไม่เรียก LLM
3. Offline Router จำแนกคำถามที่ชัดเจนด้วย keyword
4. ถ้า Offline Router ได้ `general` แต่ Offline Gate ยืนยันว่าเป็นเรื่องประกัน ให้ LLM Router จำแนกอีกครั้งเพียงหนึ่งครั้ง
5. ถ้า LLM ยังได้ `general` ให้ตอบถามรายละเอียดเพิ่ม ไม่เข้า RAG
6. ถ้าได้ `car_insurance` หรือ `life_insurance` จึงเข้า RAG ตามเดิม

## การป้องกันค่าใช้จ่ายและข้อผิดพลาด

- LLM Router ตั้ง `temperature=0` และจำกัดคำตอบให้เหลือสามหมวด
- ถ้า API error, timeout หรือส่งค่าผิด ให้คืน `general` อย่างปลอดภัย
- คำถามเดิมยังใช้ cache คำตอบตามระบบเดิม
- ไม่ให้ LLM Router ตอบลูกค้า มีหน้าที่จำแนกหมวดเท่านั้น

## เทสที่ต้องมี

- คำถามรถหรือชีวิตที่ชัดเจนไม่เรียก LLM
- คำถามกว้างไม่เรียก LLM และไม่เข้า graph
- คำถาม `2+ กับ 3+` เรียก LLM Router หนึ่งครั้งและเข้า car graph
- คำถาม `ซ่อมห้างกับซ่อมอู่` ผ่าน Offline Gate และเข้า car graph
- คำตอบ LLM ผิดรูปแบบหรือ API error คืน `general`
