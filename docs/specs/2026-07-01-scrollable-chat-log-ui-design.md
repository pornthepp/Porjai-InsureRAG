# Scrollable Chat and Log UI

## เป้าหมาย

ปรับหน้า Streamlit ใน `app.py` ให้ใกล้เคียง `design/screen.png` และ `design/code.html` โดยไม่ให้ข้อความจำนวนมากดันหน้าเว็บยาวลง

## ขอบเขต

- คงสองคอลัมน์: RAG Chat ซ้าย และ Pipeline Log ขวา
- Chat header และช่องพิมพ์อยู่กับที่ ประวัติข้อความเลื่อนในกล่องได้
- Pipeline Log header อยู่กับที่ รายการขั้นตอนเลื่อนในกล่องได้
- Raw Logs เปิด/ปิดและเลื่อนเนื้อหายาวภายในได้
- ใช้สี ระยะห่าง ขอบ และมุมโค้งตามไฟล์ใน `design`
- ไม่เปลี่ยนขั้นตอน RAG, session state หรือข้อมูล

## แนวทาง

ใช้ `st.container(height=...)` กำหนดพื้นที่เลื่อนให้ Chat และ Pipeline Log โดยตรง เสริม CSS เฉพาะงานตกแต่งและ Raw Logs เพื่อหลีกเลี่ยงการผูกกับ HTML ภายในของ Streamlit

หน้าจอแคบยังเรียงคอลัมน์ตามพฤติกรรม responsive ของ Streamlit และแต่ละกล่องมีพื้นที่เลื่อนของตัวเอง

## กรณีว่างและข้อผิดพลาด

- หากยังไม่มี Log แสดงข้อความว่างในกล่องความสูงคงที่
- ข้อความหรือ Log ที่ยาวเกินพื้นที่เลื่อนแนวตั้ง โดยไม่ตัดข้อมูล

## การตรวจสอบ

- เพิ่มการทดสอบโครงสร้าง UI ก่อนแก้โค้ด
- รันชุดทดสอบเดิมทั้งหมด
- รันแอปโหมดทดสอบและตรวจหน้าจอเดสก์ท็อปกับหน้าจอแคบ

## เกณฑ์ผ่าน

- Chat และ Pipeline Log เลื่อนได้แยกจากหน้าเว็บ
- ช่องพิมพ์และข้อความใหม่ทำงานเหมือนเดิม
- Raw Logs ยาวไม่ทำให้ทั้งหน้าขยายผิดรูป
- หน้าตาโดยรวมตรงต้นแบบ และระบบตอบคำถามเดิมไม่เสีย
