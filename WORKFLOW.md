# Workflow — คุยกับน้องพอใจ ทำงานยังไง

เอกสารนี้อธิบายว่าตั้งแต่ผู้ใช้พิมพ์คำถาม จนถึงได้คำตอบ ระบบทำอะไรบ้าง
ทีละขั้นตอน ไล่ตามไฟล์จริงในโปรเจกต์

---

## ภาพรวมสั้นๆ

```
ผู้ใช้พิมพ์คำถาม (React)
        ↓
   FastAPI backend
        ↓
   chat_service.py  ← "สมองส่วนตัดสินใจ" ว่าต้องทำอะไรต่อ
        ↓
   ถ้าตอบเองได้เลย (ทักทาย/เมนู/จำได้/เคยตอบแล้ว) → ตอบทันที ไม่เรียก AI
        ↓
   ถ้าจำเป็นต้องค้นข้อมูลจริง → เข้า RAG Pipeline (LangGraph)
        ↓
   ค้นเอกสาร → ตรวจว่าเกี่ยวข้องไหม → สร้างคำตอบ → ตรวจว่าคำตอบไม่มั่ว
        ↓
   ส่งคำตอบกลับ พร้อม log ทุกขั้นตอนที่ทำไปแบบ real-time
```

หลักการสำคัญที่ยึดตลอดทั้งระบบ: **ห้ามเรียก AI ถ้าไม่จำเป็น** และ
**ห้ามตอบถ้าไม่มีข้อมูลจริงรองรับ**

---

## เดินตามตัวอย่างจริง: "ประกันรถยนต์ชั้น 1 คุ้มครองอะไรบ้าง"

### ขั้นที่ 1 — แก้คำสะกดผิด
**ไฟล์: `text_normalizer.py`**

ทุกคำถามจะถูกส่งผ่านฟังก์ชันนี้ก่อนเป็นอันดับแรก มันมี "พจนานุกรมคำผิด"
เก็บไว้กว่า 100 คำที่คนพิมพ์ผิดบ่อยๆ (เช่น "ปะกัน" → "ประกัน",
"รถยน" → "รถยนต์") ถ้าคุณพิมพ์ผิด ระบบจะแก้ให้อัตโนมัติตรงนี้
ก่อนไปทำอะไรต่อ

### ขั้นที่ 2 — เช็คว่าเป็นคำถามกว้างไปไหม
**ไฟล์: `chat_service.py`**

เช็คก่อนว่าข้อความที่พิมพ์มาเป็นคำกว้างๆ แบบ "ประกันชีวิต" เฉยๆ
(ไม่ระบุว่าอยากรู้เรื่องอะไร) ไหม ถ้าใช่ จะตอบเมนูย่อยกลับไปเลย
ไม่ต้องทำอะไรต่อ — แต่ตัวอย่างของเรา "ประกันรถยนต์ชั้น 1
คุ้มครองอะไรบ้าง" ชัดเจนอยู่แล้วว่าอยากรู้เรื่องความคุ้มครอง
เลยผ่านขั้นนี้ไป

### ขั้นที่ 3 — เช็คว่าเป็นคำทักทาย/นอกเรื่องไหม
**ไฟล์: `offline_gate.py`**

เช็คด้วย keyword list ล้วนๆ (ไม่เรียก AI เลย) ว่าข้อความนี้เกี่ยวกับ
ประกันภัยไหม คำว่า "ประกันรถยนต์" อยู่ในลิสต์คำที่รู้จักอยู่แล้ว
เลยผ่านด่านนี้ไปได้ทันที (ถ้าเป็น "สวัสดีครับ" หรือ "วันนี้ฝนตกไหม"
จะถูกตอบเองตรงนี้เลยและจบเลย ไม่ไปต่อ)

### ขั้นที่ 4 — เช็คว่าเคยถามคำถามนี้มาก่อนไหม
**ไฟล์: `chat_service.py`**

ระบบเก็บ "คำตอบที่เคยตอบไปแล้ว" ไว้ในความจำ (cache) ถ้าเคยมีคนถาม
คำถามเดียวกัน (แบบเป๊ะๆ) มาก่อนในช่วงที่แอปยังไม่ปิด จะเอาคำตอบเก่ามา
ตอบเลย ไม่ต้องคิดใหม่ — ถ้าเป็นคำถามใหม่ (แบบตัวอย่างเรา) ก็ไปขั้นต่อไป

### ขั้นที่ 5 — จัดหมวดคำถาม
**ไฟล์: `query_router.py`**

ต้องรู้ก่อนว่าคำถามนี้เกี่ยวกับ "ประกันรถยนต์" หรือ "ประกันชีวิต"
เพื่อจะได้ค้นข้อมูลถูกที่ ระบบจะลองเช็คด้วย keyword ก่อน (คำว่า
"ประกันรถยนต์" ชัดเจนอยู่แล้ว จัดหมวดได้ทันทีแบบฟรีๆ) — ถ้า keyword
ตัดสินใจไม่ได้ (เช่นคำถามกำกวม) ค่อยเรียก AI (LLM) มาช่วยตัดสิน
1 ครั้ง

### ขั้นที่ 6 — เริ่มเข้า RAG Pipeline จริงๆ
**ไฟล์: `graph_builder.py` + `graph_state.py`**

ตรงนี้คือจุดที่ `chat_service.py` ส่งคำถามเข้า "กราฟ" ที่ประกอบไว้ล่วงหน้า
(เหมือนผังงานที่บอกว่าทำอะไรก่อนหลัง) โดยมี "สถานะ" (state) ที่เก็บว่า
คำถามคืออะไร, เอกสารที่เจอมีอะไรบ้าง, ลองค้นไปกี่รอบแล้ว ฯลฯ

### ขั้นที่ 7 — Route Node
**ไฟล์: `graph_nodes.py` (route_node)**

node แรกในกราฟ ยืนยันหมวดอีกครั้ง (หรือจัดหมวดถ้ายังไม่มี)

### ขั้นที่ 8 — Search Node
**ไฟล์: `graph_nodes.py` (search_node) → `retrieval_pipeline.py`**

ค้นหาในฐานข้อมูล ChromaDB เฉพาะหมวด "ประกันรถยนต์" โดยแปลงคำถามเป็น
embedding (ตัวเลขที่แทนความหมาย) แล้วหาเอกสารที่ "ความหมายใกล้เคียง"
ที่สุด — เจอเอกสารเรื่องความคุ้มครองประกันรถยนต์ชั้น 1 พอดี

### ขั้นที่ 9 — Grade เอกสาร
**ไฟล์: `retrieval_grader.py`**

เอาเอกสารที่เจอไปให้ AI เช็คอีกทีว่า "เกี่ยวข้องกับคำถามจริงไหม"
(กันกรณี embedding หาเอกสารที่ใกล้เคียงแค่ผิวเผินแต่จริงๆ ไม่ตรงเรื่อง)
เอกสารของเราผ่านการตรวจ

> **ถ้าค้นแล้วไม่เจอเอกสารที่เกี่ยวข้องเลย** ระบบจะไปที่
> **`query_rewriter.py`** ปรับคำถามให้เป็นทางการขึ้นแล้ววนกลับไปค้นใหม่
> (ขั้นที่ 8) — ทำได้จำกัดจำนวนรอบ ถ้ายังไม่เจอจะปล่อยผ่านไปขั้นถัดไป
> แบบไม่มีเอกสารเลย (ซึ่งจะกลายเป็นคำตอบ "ไม่พบข้อมูล" ในขั้นถัดไป)

### ขั้นที่ 10 — Generate Node
**ไฟล์: `graph_nodes.py` (generate_node) → `generator.py`**

ส่งเอกสารที่ผ่านการตรวจแล้วให้ AI สร้างคำตอบ โดยมีกฎเหล็กว่า
**ต้องตอบจากเอกสารที่ให้เท่านั้น ห้ามแต่งเพิ่ม** ถ้าไม่มีเอกสารเลย
จะไม่เรียก AI ด้วยซ้ำ แค่ส่งข้อความ "ไม่พบข้อมูล" มาตรฐานกลับไปเลย

### ขั้นที่ 11 — Hallucination Check
**ไฟล์: `graph_nodes.py` (hallucination_check_node) → `hallucination_grader.py`**

ด่านสุดท้ายก่อนส่งคำตอบออกไป เช็คว่าคำตอบที่ AI เพิ่งสร้าง
"ตรงกับเอกสารจริงไหม" หรือ AI แต่งเติมอะไรเกินมา ถ้าตรง (Grounded)
→ ส่งคำตอบออก จบ

> **ถ้าไม่ตรง (Hallucinated)** ระบบจะวนกลับไปสร้างคำตอบใหม่
> (ขั้นที่ 10) ทำได้จำกัดจำนวนรอบเหมือนกัน ถ้ายังไม่ผ่านอีก
> จะใช้คำตอบ fallback มาตรฐานแทนคำตอบที่มั่วมา

### ขั้นที่ 12 — ส่งกลับไปหน้าเว็บ
**ไฟล์: `backend/main.py`**

ทุกขั้นตอนข้างบน (2-11) ที่ทำไป จะถูกดักจับเป็น log แล้วส่งกลับไปที่
React แบบ real-time ทีละบรรทัด (ไม่ต้องรอจบหมดค่อยเห็น) — นี่คือที่มา
ของ panel "กระบวนการคิด" ที่เห็นในหน้าเว็บ

## เส้นทางอื่นๆ ที่สั้นกว่า (ไม่ต้องเข้า RAG เต็มรูปแบบ)

| สถานการณ์ | จบที่ไฟล์ | เรียก AI ไหม |
|---|---|---|
| ทักทาย ("สวัสดีครับ") | `offline_gate.py` | ❌ ไม่เรียก |
| นอกเรื่อง ("วันนี้ฝนตกไหม") | `offline_gate.py` | ❌ ไม่เรียก |
| คำถามกว้าง ("ประกันชีวิต") | `chat_service.py` (เมนู) | ❌ ไม่เรียก |
| ถามซ้ำที่เคยตอบแล้ว | `chat_service.py` (cache) | ❌ ไม่เรียก |
| ถามสั้นต่อเนื่อง ("เบี้ยประกัน" หลังคุยรถยนต์) | `chat_service.py` (context) → เข้า RAG ต่อ | ✅ เรียก (เข้า RAG) |
| คำถามชัดเจนเต็มรูปแบบ | เข้า RAG ทั้งหมด (ขั้นที่ 6-11) | ✅ เรียกหลายครั้ง |

## ไฟล์เตรียมข้อมูล (ไม่เกี่ยวกับตอนแชท รันแยกต่างหาก)

**`ingest.py`** — อ่านไฟล์ `data/car_insurance.txt` และ
`data/life_insurance.txt` แบ่งเป็นชิ้นย่อย (chunk) แปลงเป็น embedding
แล้วบันทึกลง ChromaDB

**ต้องรันคำสั่งนี้ใหม่ทุกครั้งที่แก้ไฟล์ข้อมูลใน `data/`:**
```
python ingest.py
```
ไม่งั้นข้อมูลใหม่ที่แก้ไปจะยังไม่ถูกนำมาใช้ค้นหา (backend จะ
auto-recover เองไม่ต้อง restart แล้ว หลังจากรัน ingest.py)

## แผนที่ไฟล์ทั้งหมด (สรุปตาราง)

| ไฟล์ | หน้าที่ 1 ประโยค |
|---|---|
| `text_normalizer.py` | แก้คำสะกดผิด |
| `offline_gate.py` | กรองทักทาย/นอกเรื่อง (ไม่เรียก AI) |
| `chat_service.py` | สมองตัดสินใจหลัก จำบทสนทนา จัดการ cache |
| `query_router.py` | จัดหมวดรถยนต์/ชีวิต |
| `graph_state.py` | นิยามข้อมูลที่ไหลผ่านกราฟ |
| `graph_builder.py` | ประกอบกราฟ (ผังงาน) |
| `graph_nodes.py` | node แต่ละอันในกราฟ |
| `retrieval_pipeline.py` | ค้นเอกสารจาก ChromaDB |
| `retrieval_grader.py` | ตรวจว่าเอกสารที่เจอเกี่ยวข้องไหม |
| `query_rewriter.py` | ปรับคำถามใหม่เมื่อค้นไม่เจอ |
| `generator.py` | สร้างคำตอบจากเอกสาร |
| `hallucination_grader.py` | ตรวจว่าคำตอบมั่วไหม |
| `ingest.py` | เตรียมข้อมูลลง ChromaDB (รันแยก) |
| `backend/main.py` | API server เชื่อม React กับทุกไฟล์ข้างบน |
